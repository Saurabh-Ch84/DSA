### ğŸ§© Structure of a JWT
A JWT consists of **three parts**, separated by dots (`.`):

```
Header.Payload.Signature
```

#### 1. **Header**
- Specifies metadata: the signing algorithm and token type.
```json
{
  "alg": "HS256",
  "typ": "JWT"
}
```
Encoded using Base64Url.

#### 2. **Payload**
- Contains **claims**â€”statements about the user and token metadata.
```json
{
  "sub": "1234567890",
  "name": "Saurabh",
  "iat": 1516239022,
  "exp": 1516242622
}
```
Common claims:
- `sub`: Subject (user ID)
- `iat`: Issued At
- `exp`: Expiration
- `iss`: Issuer
- `aud`: Audience

#### 3. **Signature**
- Ensures integrity and authenticity.
- Created by hashing the encoded header and payload with a secret key using the specified algorithm:
```
HMACSHA256(
  base64UrlEncode(header) + "." + base64UrlEncode(payload),
  secret
)
```
